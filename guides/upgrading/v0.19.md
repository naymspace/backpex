# Upgrading to v0.19

## Bump Your Deps

Update Backpex to the latest version:

```elixir
defp deps do
  [
    {:backpex, "~> 0.19.0"}
  ]
end
```

## Collapsible Sidebar

The `app_shell` component now ships with a unified collapsible sidebar that replaces the previous split between a mobile drawer and a desktop-only sidebar. **This introduces breaking changes to the sidebar API:**

### `topbar_branding` renamed to `sidebar_branding` and moved into `<:sidebar>`

The branding component has been renamed and belongs inside the sidebar slot, not the topbar.

**Before:**

```heex
<Backpex.HTML.Layout.app_shell ...>
  <:topbar>
    <Backpex.HTML.Layout.topbar_branding />
    <!-- ...other topbar content... -->
  </:topbar>
  <:sidebar>
    <!-- nav items -->
  </:sidebar>
</Backpex.HTML.Layout.app_shell>
```

**After:**

```heex
<Backpex.HTML.Layout.app_shell ...>
  <:topbar>
    <div class="flex-1"></div>
    <!-- ...other topbar content... -->
  </:topbar>
  <:sidebar>
    <Backpex.HTML.Layout.sidebar_branding />
    <div class="menu w-full flex-1 overflow-y-auto px-2 py-2">
      <ul class="w-full">
        <!-- nav items -->
      </ul>
    </div>
  </:sidebar>
</Backpex.HTML.Layout.app_shell>
```

### The `<:sidebar>` slot no longer auto-wraps in `<ul>`

Previously `app_shell` wrapped sidebar slot content in a `<ul>`. It no longer does. `sidebar_item` and `sidebar_section` emit `<li>` elements, so consumers must now provide their own `<ul>` wrapper (as shown above). Without the wrapper you will produce invalid HTML and lose list semantics for screen readers. The outer `app_shell` sidebar is now a `<nav>` landmark, so avoid nesting another `<nav>` inside the slot.

### Define the `--sidebar-width` CSS variable

The sidebar uses a `--sidebar-width` CSS custom property for its width and the main-content shift. Add a definition to your stylesheet:

```css
/* assets/css/app.css */
:root {
  --sidebar-width: 16rem;
}
```

### `sidebar_section` requires an explicit `id`

`Backpex.HTML.Layout.sidebar_section` no longer defaults `id` to `"section"`. The id is used as the localStorage key for the collapsed state and as the basis for `aria-controls`, so two sections without explicit ids would collide and toggle together. Every `sidebar_section` now needs its own unique id:

```heex
<Backpex.HTML.Layout.sidebar_section id="blog">
  <:label>Blog</:label>
  <!-- items -->
</Backpex.HTML.Layout.sidebar_section>
```

### JavaScript hook renamed

The `BackpexSidebarSections` hook has been replaced by `BackpexSidebar`, which now manages both the open/close state and sub-section expand/collapse. If you register hooks individually rather than spreading `BackpexHooks`, update the reference accordingly. If you use the default spread in your `app.js`, no change is required.
